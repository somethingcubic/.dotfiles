# Context Recap

恢复当前项目的工作上下文，让你快速接上之前的进度。

## 执行步骤

按顺序读取以下信息源，跳过不存在的：

### 1. TASK.md（项目进度）

读取项目根目录的 `TASK.md`，提取：
- 当前目标
- 已完成项
- 进行中的工作
- 下一步计划
- 关键决策和未解决问题

### 2. Auto Memory（行为偏好和教训）

读取 `~/.claude/projects/` 下与当前项目对应的 `memory/` 目录，提取关键记忆条目。

### 3. 最近 Git 历史

运行 `git log --oneline -10` 查看最近的改动方向。

### 4. 相关 Postmortem（踩坑记录）

扫描 `~/.claude/postmortems/` 目录，查找与当前项目相关的复盘记录，提取仍然适用的教训。

## 输出格式

```
## 上下文恢复

### 项目进度
<从 TASK.md 提取>

### 最近改动
<从 git log 提取，一句话概括方向>

### 关键决策 & 约束
<从 TASK.md 和 memory 提取>

### 注意事项
<从 postmortem 和 memory 提取，仍需注意的坑>

### 建议下一步
<基于以上信息的行动建议>
```

## 规则

1. 纯读取，不修改任何文件
2. 信息源不存在就跳过，不报错
3. 输出简洁，每个部分不超过 5 行
4. 如果所有信息源都不存在，提示"当前项目没有可恢复的上下文"
